Customizable maps can be created by utilizing the nyc-lib npm library.

To get started, include the following links and scripts in your HTML file:

                    
<link rel="stylesheet" href="https://use.fontawesome.com/releases/v5.6.3/css/all.css" integrity="sha384-UHRtZLI+pbxtHCWp1t77Bi1L4ZtiqrqD80Kn4Z8NTSRyMA2Fd33n5dQ8lWUE00s/" crossorigin="anonymous">
<link href="https://maps.nyc.gov/nyc-lib/v1.2.37/css/nyc-basic-lib.css" rel="stylesheet">
<link href="https://cityofnewyork.github.io/nyc-core-framework/css/theme.css" rel="stylesheet">
<script src="http://code.jquery.com/jquery-3.3.1.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/proj4js/2.4.4/proj4.js"></script>
<script src="https://cdn.rawgit.com/openlayers/openlayers.github.io/master/en/v5.3.0/build/ol.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/PapaParse/4.4.0/papaparse.min.js"></script>
<script src="https://maps.nyc.gov/nyc-lib/v1.2.37/js/babel-polyfill.js"></script>
<script src="https://maps.nyc.gov/nyc-lib/v1.2.37/js/nyc-ol-lib.js"></script>

Create a div to contain your map and give it an id as such:

<div id='map'></div>

The nyc.ol.FrameworkMap commonly takes the following constructor options:

  • {string} mapTarget (required) - The target element for the map. This will be the same as the id we created above.
  • {string} geoclientUrl (required) - The URL for the Geoclient geocoder with appropiate keys. If you wish to search for locations on the map, this must not be null. Request an app_id and app_key from the NYC DoITT GIS Dept.
  • {string} searchTarget (optional) - The target element for search input. The map comes with a built in search element(in the upper left corner of the map), but if you prefer it be located outside the map(or simply with your own style), provide your own search element.
  • {string} locationMarkerUrl (optional) - The URL to an image for use as the search location symbol.

    • If you plan to load facility data:

  • {string} facilityUrl (required) - The URL for the facility features data. If you plan to load data into markers and/or display them in a list, this argument must not be null.
  • {string} listTarget (optional) - The target element for facility list. If you plan to display a list of facilities, create a div element like you did with the map and include an id for the list, and pass it to the constructor.
  • {string} facilityType (optional) - The title for the facilites list. This will be 'facilities' by default.
  • {string} facilityMarkerUrl (optional) - The URL to an image for use as a facility symbol
  • {Array<number>} facilityMarkerColor (optional) - An RGB color for use as a facility symbol

    • Please check the documentation for the full list of options:

    Consider the following example:

            
var podMap = new nyc.ol.FrameworkMap({
mapTarget: '#pod-map', // 
searchTarget: '#pod-map-search',
listTarget: '#list',
facilityType: 'Points of Distribution',
facilityUrl: 'data.csv',
mapMarkerUrl: 'map-marker.png',
geoclientUrl: 'https://maps.nyc.gov/geoclient/v1/search.json?app_key=#&app_id=#'
});
