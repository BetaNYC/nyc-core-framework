Buttons and Links

Global Hover and Focus

Buttons and link :hover and :focus states underline text to ensure a noticable state-change when navigating or tabbing through content. This customization is found in scss/_d-enhancements.scss.

Disabled

Buttons and links with a disabled class include the css rule cursor: not-allowed;, which on hover, indicates that the button is disabled.

Icons Buttons

Icon buttons are created using the free version of Font Awesome. Add a no-btn-style class to <button>. When using icons without concurrent text, add a title attribute to describe the icon.

Instances of Font Awesome icons should include a role="img" attribute.

Icons inside buttons and links

To use a Font Awesome icon with any <button> or <a> element, simply add a has-icon class to the parent element. Wrap the button text with <span class="title mx-1"> with two classes: title and mx-1.

  • Link

The title class ensures only the text will be underlined on hover and focus, while the mx-1 class provides even spacing around the text when a right-to-left language is being used.

Links within articles

For accessibility, all links without a class (unstyled) that are present within #primary-content have text-decoration:underline; applied. This customization is found in scss/_e-supplementary-classes.scss.
