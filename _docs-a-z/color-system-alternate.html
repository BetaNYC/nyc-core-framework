The NYC Core Framework's dymnamic color system allows you to intuitively manage colors and apply them with little effort across pages, components, and content.

Color Accessibility

When selecting colors for your theme, it is wise to check that your colors meet a sufficient contrast ratio.

Theme Colors

The following palette makes up our default theme colors.

Modify these colors with custom properties or create additional colors using Sass. These theme colors generate our background color classes (below), text color classes, and border classes.

Background Color Classes

In the NYC Core Framework white is treated as a color and it serves as your default theme color. Throught our templates and examples, you may have noticed the <body> tag has a .bc-white class applied (Figure 1).

Each theme color (e.g. primary) is assigned a predesignated text color , link color, and border color. This predesignation occurs in our Sass partial for colors, found in /src/partials/_color.scss.

Figure 2 (below) represents the custom properties (variables) located at the top of this partial.

When a background color class is applied to an element, its text, borders, and links (only those inside <p> tags) automatically inherit their appropriate predesignated color.

Inheritance and Selectivity

Apply a background color class to any element along the DOM (Document Object Model) and every child element within that selector will inherit their designated theme color properties until a new class is specified.

In other words, background color classes are infinitely nestable. Let's look at the concept of nesting in practice.
