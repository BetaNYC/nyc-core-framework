Predefined text and icon classes for a variety of distinct, actionable button styles.

Button Accessibility

Like our navigation links, buttons provide visible hover and focus states to communicate a button's current state to the user. Icon-only buttons provide text for screen readers.

The Basic Button

All buttons on in the NYC Core Framework require the definer class .btn on an <a> or <button> tag. By default, .btn selectors inherit text color and font size from their parent. The example below is a generic button with no color or style applied.

Adding Color

Add color to a .btn selector with our predefined background-color classes. These classes automatically apply the button's appropriate text color.

Full-width Buttons

For full-width buttons, add the display utility class .d-block.

Outlined Buttons

For a button with an outline instead of a fill, add the outline modifier class .btn--outline to the .btn selector. The outline width matches your theme's default border width. The outline color inherits the button's text color, which can be changed using our text color utility classes.

Border Radius

Use our prebuilt border-radius utility classes to add rounded edges to a .btn selector.

Button Sizes

Instead of providing predetermined button sizes, Framework buttons inherit their font size and use relative (em) sizes for padding. This means you can make whatever size button you wish, using our prebuilt font size classes.

Whn you have multiple buttons within the same selector, you won't need to specify a size for each button. In the example below, the predefined small font size (class .fs-sm) is applied to a list of buttons, making all buttons smaller with a single class.

Buttons With Text and Icon

Add an icon to a .btn selector with the .btn--has-icon modifier class. Buttons with icons require two child selectors. The icon itself is a <span> with the combination Font Awesome classes, with an additional class of .btn__icon.

A second <span> with a .btn__text class is needed for the button's text. This selector is our target for underlining the text on :hover and :focus states. It also provides a small margin from the icon.

If you wish for a button's text and icon to be opposite of one another, add the chain modifier class --disperse to .btn--has-icon class (resulting in .btn--has-icon--disperse). This class automatically makes the button full-width of its parent.

Icon-only Buttons

To create an accessible icon-only button, add the .btn--icon-only modifier class to a .btn selector. Include the title attribute to describe the button on its hover state (for sighted users).

Like our buttons with text and icon (above), icon-only buttons also require two child selectors. A <span> with an .sr-only class that includes descriptive text for screan readers, and another <span> selector with the combination Font Awesome classes.

Icon Buttons are transparent by default. Just like regular buttons, they work well with our background-color classes.

You may also use our text color classes for even more color combinations.
