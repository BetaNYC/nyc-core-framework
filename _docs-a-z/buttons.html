The Button Selector

Buttons within the NYC Core Framework require a .button utility class on an <a> or <button> tag (Figure 1 below). By default, .button selectors inherit text color and font size from their parent. The example below is a generic .button selector with no other styles yet applied.

Coloring Buttons

Add color to a .button selector with our theme-color utilities. These utilities apply the button's appropriate text color automatically (see Figure 3).

Full-width Buttons

For buttons that span the width of their parent, add the .width-100 size utility class (Figure 3).

Do not use .display-block for full-width buttons. Buttons have a display property of flex (for vertical centering icons). Using use .display-block changes the buttons display property from flex to block. This breaks the vertical centering of the icon within the button (See buttons with text and icon below).

Outlined Buttons

Create a button with an outline instead of a fill. Add the outline modifier class .button--outline to the .button selector. The outline width matches your theme's body border width. The outline color inherits the button's text color. You can change the color using our text color utility classes.

Button Border Radius

Use our border-radius utility classes (e.g. .rounded) for rounded edges on .button selectors.

Button Sizes

Instead of providing predetermined button sizes (like you'll find in other frameworks) our buttons inherit their font size and use relative (em) sizes for padding, allowing buttons to be resized using our font size utility classes.

This is particularly useful because multiple buttons can be resized within a single selector. In the example below, the small font size utility class (.font-size-sm) is applied to <ul> containing buttons, making all buttons smaller with a single class.

Buttons With Text and Icon

Add an icon to a .button selector with the .button--has-icon modifier class. Buttons with icons require two child selectors. The icon itself is a <span> with its combined Font Awesome classes, and an additional class of .button__icon.

A second <span> with a .button__text class is needed for the button's text. This selector is our target for underlining the text on the button's :hover and :focus states. It also provides a small margin to separate the text from the icon.

If you wish for a button's text and icon to be opposite of one another, add the chain modifier class --disperse to .button--has-icon class (resulting in .button--has-icon--disperse). This class automatically makes the button full-width of its parent.

Icon-only Buttons

Icon-only buttons require the .button definer class and the .button--icon-only modifier class. Include the title attribute to describe the button on its hover state (for sighted users).

Like our buttons with text and icon (above), icon-only buttons also require two child selectors. A <span> with an .sr-only class that includes descriptive text for screan readers, and another <span> selector with the combination Font Awesome classes.

Icon-only buttons are transparent by default. Like our other buttons, color can be applied with background-color and text color utility classes.
