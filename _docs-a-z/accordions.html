Accordions are most practical when only specific content might be relevant to a visitor—like an FAQ page, or when several topics are covered on a single page. It's best to avoid using accordions to simply make long pages appear shorter.

Accordion Anatomy

The accordion component is made up of a parent .accordion selector and two required child selectors: .accordion__button and it's subsequent sibling .accordion__panel. These children depend on one another as a pair. Each pair makes up a single topic within their parent accordion.

Accordions have one granchild: the .accordion__panel__content selector (found within .accordion__panel). Lorem ipsum dolor sit amet consectetur adipisicing elit.

Be aware that custom padding directly applied to the .accordion__panel selector will not work. If you wish to apply custom padding to the accordion panel, either with CSS or one of our predefined padding classes, you may do so on the .accordion__panel selector's child <div>, which includes the data-accordion="content" attribute (see Figure 1, line 8).

Semantic Headers

Semantically, accordion headers should match the information architecture of the page. For example, if an accordion group lives directly below a page's <h1> header tag, the accordion headers should be <h2> tags, or have an aria-level="2" attribute present.
