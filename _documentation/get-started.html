Make beautiful, accessible websites easier and faster.

The NYC Core Framework is an open-source front end toolkit for building modern websites and browser-based applications. Our lightweight codebase is easy to understand—and at the same time—provides a wealth of accessible components ideal for projects of any size and complexity.

Quick Start

Get up and running in a New York minute. Basic knowledge of HTML and CSS is all you'll need.

CSS and Javascript

The NYC Core Framework relies on only four dependencies. Within the <head> tag of index.html, you will find a stylesheet <link> to Font Awesome, an open-source icon library, and another <link> for our theme's principle stylesheet, with the path /css/theme.css.

Near the closing <body> tag, we link to our theme's javascript file, with the path /js/main.js.

And finally, also near the closing <body> tag, a polyfill for Internet Explorer 11 is included. This dependency is required only if you wish to support this last version of Internet Explorer.

Customizing with Sass

The NYC Core Framework is customizable using Sass (SCSS). The theme's stylesheet, /css/theme.css is generated (or compiled) from the central SASS file, with the path /scss/theme.scss.

At the top of theme.scss in Section A. SCSS Imports, all dependent SASS partials are imported. These partials are located in the /scss/base/ directory. They work together and act as building blocks for styling our theme.

Some partials are required, while others are optional—and can be safely commented out or altogether removed.

For your convenience, SASS files in the NYC Core Framework are divided into alphabetically labeled Sections, with an In this File key at the very top. Throughout the documentation, we'll often refer to a section (e.g., Section A.) within a partial, when describing a feature or customization.

Adding your own CSS

We recommend adding your own CSS or Sass under Section B. of /scss/theme.scss. This way your customizations are in one place and remain separated from the building blocks (SASS partials) of the framework.

Browser Support

In the spirit of inclusivity, we aim to support all modern browsers—including Internet Explorer 11—which still today is widely used in government agencies, non-governmental organizations, and grandparents' houses.

Note that proxy browsers (such as Opera Mini, Opera Mobile’s Turbo mode, UC Browser Mini, Amazon Silk) have not been tested.
