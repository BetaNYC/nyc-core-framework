Distinguish yourself without starting from scratch.

The NYC Core Framwork makes it easy to apply your own color combinations, typography, spacing, and sizing to our prebuilt layouts and components.

While it's possible to customize NYC Core Framework using Sass, since version 2.0, we offer a more convenient approach to theming with Custom Properties (also known as CSS variables).

Custom Properties

Custom properties are similar to Sass variables because they both allow you to store reusable values.

Custom properties, however, provide additional benefits that Sass variables do not. Custom properties are native CSS syntax. This means you do not need to set up a Sass development environment to modify them. Additionally, custom properties can be scoped globally to the HTML (using the :root pseudo-class) and scoped locally to a selector (e.g. .selector--primary).

Custom properties can be scoped globally and redefined locally. Consider the following example where --primary is defined on :root, and redefined on .selector--primary.

In the NYC Core Framework, the color palette, font styles, spacing utilities, and other design characteristics are declared using custom properties.
