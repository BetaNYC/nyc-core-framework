Custom Properties

It is possible to personalize NYC Core Framework using Sass, however, since version 2.0, we offer a more convenient methodology with Custom Properties.

Custom properties (also known as CSS variables) are similar to Sass variables because they both allow you to store reusable values. Custom properties, however, provide additional benefits not offered by Sass variables.

Custom properties are native CSS syntax. This means you do not need to set up a Sass development environment to modify them. Additionally, custom properties can be scoped globally to the HTML (using the :root pseudo-class) and scoped locally to a selector (e.g. .selector--primary).

In the NYC Core Framework, the color palette, font styles, spacing utilities, and other design characteristics are declared using custom properties.

Customize Your Theme Using a Second Stylesheet

Within your project, create a new CSS file and name it theme-properties.css and save it to the /css/lib/ directory.

Next, open your HTML document and add a link to theme-properties.css below nyc_core.css.
