A panorama of possibilities.

Backdrop components place a translucent image behind any custom content. Backdrops offer two distinct variations: an image-proportional backdrop that scales and stacks responsively, and a fixed-height backdrop that can adjust to the height of its inner content.

Image Accessibility

Backdrops replace CSS background images with inline HTML images so screen readers can read the image's alt text.

Learn more about accessible images.

Image-proportional

The default .backdrop selector requires two immediate children. The first is an image with a class of .backdrop__image. This image sets the size (proportional width and height) of our backdrop. The second direct child is a block-level element with a class of .backdrop__cover, inside of which custom content can be placed.

Examples on this page include the .backdrop__image-credit selector which is not required and can be customized in /scss/base/_backdrops.scss.

Background-color and Image Opacity Classes

Backdrops utilize two other Natura11y Inclusive Framework features: Background-color and Opacity classes. The opacity class (.opacity-40--lg) applies a CSS opacity to the image, allowing the background color (set with .bc-dark) to show through.

Did you see that opacity classes have their own breakpoint modifiers? If present, the opacity will be set only at the given breakpoint.

In the example above, we're telling .backdrop to position the .cover absolute over the .image at the large breakpoint. Also at the large breakpoint, a 40% opacity is applied to the image.
